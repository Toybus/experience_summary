# 隐性经验成型 Skill 设计

## 1. 背景

目标用户是工作约一年的 junior 同学。她能够产出较高质量的调研报告、分析文档和产品功能文档，但在团队分享时，往往只能复述文档内容，难以解释自己如何做出同等质量的产出。

问题不在于她没有经验，而在于部分经验尚未被自己意识到，或仍以模糊直觉存在。Skill 需要阅读作者的一篇优秀文档，通过自适应访谈帮助作者把隐性经验推进到当前证据允许的最清晰状态，再将经验嵌入原文分享内容，使作者本人和听众都能获得收益。

## 2. 目标与非目标

### 2.1 目标

Skill 应当：

1. 从原文、作者输入和外部反馈中识别值得深挖的经验线索。
2. 沿作者的思路进行多轮访谈，避免机械切换话题。
3. 允许经验保持不同成熟度，不强迫模糊直觉形成完整方法论。
4. 保留作者原文和访谈中的语言风格，只将内容抽象一层。
5. 产出作者经验资产，帮助作者认识自己的做法和判断。
6. 以原文讲解为主体，将经验与读者应用自然嵌入相应章节。
7. 让听众不仅理解作者如何思考，也知道自己在什么情境下可以怎样尝试。

### 2.2 非目标

Skill 不负责：

- 生成完整逐字稿；
- 制作幻灯片；
- 将原文重写成一篇新报告；
- 交付后的多轮润色和修改；
- 把所有直觉包装成成熟、专业的方法论。

## 3. 核心原则

### 3.1 成型不等于强行闭环

Skill 的职责是把经验推进到当前证据允许的最清晰状态。作者无法解释的部分可以保留为工作假设或待探索直觉。完整性不能凌驾于真实性。

### 3.2 语言不拔高，认知抬一层

表达尽量使用作者原文和访谈中的词汇、句式与叙述习惯。抽象只指出其中可重复的判断或动作，不包装成作者自己无法自然讲出口的专业体系。

### 3.3 原文、经验与读者收益耦合

分享主体是原文的结果和推理。经验应出现在最相关的原文节点旁，随后转化为适合该经验的读者应用方式。只有发生在写作前、影响全文或横跨多个章节的高层经验，才单独放在开场或结尾。

### 3.4 由证据约束抽象

原文事实、作者访谈、外部反馈和 Skill 推断必须保留来源。Skill 的推断只能用于提出问题，不能未经作者确认就成为作者经验。

## 4. 输入

### 4.1 必需输入

- 一篇作者本人撰写、且被认为质量较好的文档。

### 4.2 可选输入

- 作者主动指出的重点；
- 老板、同事或评审者给出的反馈；
- 分享对象、分享时长等限制。

### 4.3 材料读取要求

Skill 必须理解承载关键结论的完整材料，而不只读取目录或正文文本。

对于飞书文档，应读取：

- 正文和标题结构；
- 关键画板；
- 嵌入电子表格；
- 对结论有影响的图片和附件；
- 其他被正文引用、且理解推理链所必需的内容。

如果某种嵌入内容无法读取，Skill 必须说明遗漏、可能影响和当前分析边界，不能假装已经分析。

## 5. 整体用户旅程

### 5.1 接收材料

读取文档及必要的嵌入内容，并接收作者主动输入和外部反馈。外部反馈是发现经验的探针，不直接等同于经验。

### 5.2 建立动态经验地图

候选节点来自三类信号：

1. 原文体现出的高质量判断、结构或做法；
2. 作者主动提供的线索；
3. 外部反馈指出的亮点。

每个节点记录：

- 对应的原文位置或证据；
- 值得深挖的原因；
- 线索来源；
- 当前状态：未探索、探索中、已成型、保留为直觉或跳过。

Skill 向作者展示一张精简地图，并邀请作者补充或调整。地图是导航工具，不是固定访谈提纲。

### 5.3 顺势访谈

访谈采用动态的深度优先方式：

- 当前回答带出相关经验时，沿作者思路继续；
- 新线索可以新增、合并或重排节点；
- 当前线索停滞、重复、被跳过或已经成型时，再返回地图；
- 一次只问一个主要问题。

### 5.4 轻量透传进度

Skill 在自然转场时说明当前方向和原因，例如：

- “这条经验已经比较清楚。”
- “刚才的回答带出了一个更相关的方向，我先顺着它继续。”
- “这一部分开始重复，我建议回到另外一个节点。”
- “目前核心模式已经出现，可以开始收束。”

不固定频繁显示数字进度。作者主动询问进度时，才展示更完整的地图状态。访谈中段提供一次继续、调整优先级或提前收束的机会。

### 5.5 饱和判断与收束

出现以下信号时，Skill 主动建议收束：

- 新回答开始重复已有模式；
- 连续追问没有产生新信息；
- 优先节点已经达到作者当前能表达的清晰度；
- 作者明显希望停止。

Skill 只提出建议，最终由作者决定继续或结束。

### 5.6 生成成果

Skill 生成两层成果：

1. 作者经验资产；
2. 以原文为主体的耦合式分享内容。

## 6. 访谈与经验成型机制

### 6.1 证据账本

在生成方法论之前，Skill 记录：

- 原文片段或位置；
- 作者访谈原话；
- 外部反馈；
- Skill 暂时提出的推断。

每条经验必须能够追溯到证据账本。

### 6.2 问题选择

根据当前信息选择最自然的一种追问：

- 过程还原：“你当时先做了什么，接着发生了什么？”
- 决策追问：“为什么先回答这个问题，而不是直接做方案？”
- 对比追问：“如果不这样做，另一种做法会带来什么？”
- 证据追问：“你用什么材料判断这个结论成立？”
- 来源追问：“这是你原本的判断、外部反馈，还是 AI 帮你发现的？”
- 边界追问：“这条经验适用于什么情况，不适用于什么情况？”

问题优先承接作者刚刚提供的新信息，不按模板轮询所有问题类型。

### 6.3 临时复述与成型门槛

Skill 可以临时复述当前理解，但必须允许作者修正。满足以下至少一个条件时，经验才进入“已成型”：

- 有具体案例，且作者确认了其中的做法或判断；
- 在多个回答中重复出现；
- 作者能够说明它如何影响实际决策。

不满足时保留为工作假设或待探索直觉。

### 6.4 作者卡住时的处理

作者说“很难回答”“更像直觉”时：

1. 停止用同一种方式继续追问；
2. 根据已有证据提出 2–3 个低强度解释；
3. 标明这些解释只是供作者辨认的假设；
4. 允许多选、修改、全部否定或跳过；
5. 跳过后保留原始表述，不将候选解释写成作者结论。

### 6.5 经验成熟度

- **已验证经验**：有明确案例、做法和决策影响。
- **工作假设**：符合当前案例，但尚不知道能否稳定复现。
- **待探索直觉**：作者认为它重要，但暂时无法解释。
- **跳过**：作者不希望继续，或当前没有足够信息。

成熟度用于保护真实性，不是要求逐级升级的任务清单。

## 7. 成果结构

### 7.1 作者经验资产

每条经验按实际成熟度选用以下字段：

- 贴近作者语气的经验表述；
- 对应的原文和访谈依据；
- 当前成熟度；
- 比作者原话高一层的抽象；
- 尚未想清楚的部分；
- 适合迁移时，读者可以如何使用。

字段不是强制填满。直觉可以只保留作者原话、相关案例和待观察问题。

### 7.2 耦合式分享内容

以原文顺序组织分享，每个片段按需要组合：

1. **原文内容**：展示或精简讲解什么，引用哪张表、画板或结论。
2. **作者讲述提示**：用作者熟悉的语言说明得出了什么，不写逐字稿。
3. **经验穿插**：补充作者当时如何思考、判断或调研。
4. **读者应用**：选择适合该经验的形式，帮助听众迁移。

### 7.3 高层经验

发生在写作前、影响全文或横跨多个章节的经验可以：

- 在开场简短说明；
- 在多个原文章节中展示其具体作用；
- 在结尾归纳。

不强行绑定到单个段落，也不另写成脱离案例的方法论课程。

### 7.4 读者应用形式

Skill 根据经验类型选择表达方式：

- 判断类经验：读者可以问自己的问题；
- 操作类经验：可以立即尝试的动作；
- 证据类经验：简单核验方法；
- 风险类经验：容易忽略的信号；
- 取舍类经验：比较维度；
- 工作假设：邀请读者在自己的案例中观察，不写成确定规则。

## 8. 异常与边界处理

### 8.1 文档无法读取

说明缺失，并请作者提供可访问链接、导出文件或正文。不能根据标题继续推测。

### 8.2 文档很长

先读取目录和结构，再按章节精读。优先读取与候选节点相关的章节，同时覆盖所有关键结论。不能把每个章节机械转换成经验节点。

### 8.3 经验信号很少

明确说明当前材料只支持少量经验，不为了丰富输出而过度抽象。可以建议补充另一篇文档，但仍完成已有内容。

### 8.4 作者回答矛盾

把矛盾作为探索线索，询问不同表述是否发生在不同阶段。无法确定时保留多个版本和不确定性。

### 8.5 作者持续卡住

记录原始直觉并停止深挖。不能无限更换问法，也不能由 Skill 替作者完成答案。

### 8.6 没有外部反馈

继续从原文和作者输入生成地图，不将其视为阻塞。

## 9. 质量标准

### 9.1 材料理解

- 覆盖正文及影响结论的画板、表格、图片和附件；
- 区分原文事实、作者解释、外部反馈和 Skill 推断；
- 每个候选节点可以追溯到原始材料。

### 9.2 访谈

- 一次只问一个主要问题；
- 优先沿作者当前思路追问；
- 轻量透传方向和进度；
- 作者卡住时提供候选解释和跳过选项；
- 不以填满模板为结束条件；
- 能识别信息饱和并建议收束。

### 9.3 经验总结

- 作者认为内容符合自己的真实思路和说话方式；
- 抽象比原话高一层，但没有构造虚假的成熟体系；
- 已验证经验、工作假设和直觉得以区分；
- 重要经验保留原文和访谈依据；
- 不完整经验可以保持不完整。

### 9.4 分享内容

- 主体仍是原文结果与推理；
- 经验出现在最相关的原文节点旁；
- 只有高层、跨章节经验被单独抽出；
- 可迁移经验被转化为合适的读者应用；
- 讲述提示贴近作者语气，不生成完整逐字稿。

### 9.5 成功判断

一次执行成功需要作者同时确认：

1. **忠实性**：“这符合我的真实思路和说话方式。”
2. **可迁移性**：“团队成员听完后知道自己在什么情况下可以怎样尝试。”

只满足忠实性，说明完成了复盘，但尚未完成分享。

## 10. 验证设计

### 10.1 第一组真实样本

使用 Webform 调研文档及本轮访谈记录作为首个真实样本。测试材料保留原始文档和原始回答，不向受测 agent 泄露本设计的预期答案。

### 10.2 RED：无 Skill 基线

让新的 agent 在没有本 Skill 的情况下完成同类任务，记录实际行为，重点观察是否：

- 直接总结文档，没有真正访谈；
- 使用固定模板强行补齐经验；
- 过早替作者抽象；
- 访谈过长且不透传进度；
- 单独生成分享框架，没有与原文耦合；
- 只复现作者思路，缺少读者应用；
- 忽略画板、表格等嵌入内容；
- 把 agent 推断写成作者结论。

只记录真实出现的失败，不假设所有问题都会发生。

### 10.3 GREEN：最小 Skill

只针对 RED 中实际出现的问题编写最小 Skill，再用相同材料和等价访谈回答验证：

- 是否形成动态经验地图；
- 是否顺着作者思路访谈；
- 是否保留模糊直觉和跳过权；
- 是否产出耦合式分享内容；
- 是否提供可使用的读者启发。

### 10.4 REFACTOR：变体测试

使用以下变体寻找新问题：

1. 作者频繁回答“就是直觉”；
2. 外部反馈与原文证据不完全一致；
3. 主要价值藏在画板和表格中；
4. 文档很长，但值得分享的经验只有两三条；
5. 作者经验丰富，但其中部分不适合迁移。

发现新误判后，只增加解决该误判所需的最小规则。

### 10.5 措辞微测试

对影响行为的关键措辞进行多次独立测试，始终包含无指导对照。重点比较：

- 固定模板与成熟度描述；
- 普通追问与“候选解释 + 跳过”；
- 数字进度与轻量进度透传；
- 独立分享框架与耦合式分享内容。

每个关键措辞变体至少运行五次独立样本，人工阅读所有结果，并把输出一致性作为指标。

## 11. 首个真实样本中已识别的经验方向

以下方向用于验证 Skill 能否发现经验，不作为硬编码的预期答案：

- 先判断“做不做”，再讨论“怎么做”；
- 不把业务目标等同于默认方案；
- 对可能反复出现的备选方案做二次辨析；
- 小样本负责看懂，大样本负责判断信号强弱；
- 先形成差异假设，再用案例补充和校准；
- 从“某个模块很重要”追问其真正解决的判断问题；
- 从复用假设出发识别差异并划分系统边界；
- 作者定方向、AI 扩展分析、原始证据校准。

测试成功不要求逐字复现这份列表，而要求结果忠实、有证据、不过度抽象，并能转化为读者可使用的启发。
